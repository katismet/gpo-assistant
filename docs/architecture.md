# Архитектура системы ГПО-Помощник

```
┌──────────┐
│ Telegram │
│   Bot    │
└────┬─────┘
     │
     │ HTTP/Webhook
     │
     ▼
┌─────────────────┐
│  Backend Service │
│   (Python/AIO)   │
└────┬─────────┬───┘
     │         │
     │         │
     ▼         ▼
┌─────────┐  ┌──────────┐
│Bitrix24 │  │AI Module │
│  REST   │  │  (LLM)   │
│   API   │  └──────────┘
└─────────┘
```

## Компоненты

- **Telegram Bot** - интерфейс взаимодействия с пользователями
- **Backend Service** - основной сервис обработки запросов
- **Bitrix24 REST API** - интеграция с CRM системой
- **AI Module (LLM)** - анализ данных и генерация инсайтов

## Поток данных

1. Пользователь взаимодействует с ботом через Telegram
2. Backend Service обрабатывает запросы и управляет FSM состояниями
3. Данные синхронизируются с Bitrix24 через REST API
4. AI Module анализирует данные и генерирует управленческие инсайты
