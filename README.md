1) Название проекта:
ГПО-Помощник — AI-система управления строительными сменами

2) Для кого (роль/компания):
Прорабы, начальники участков, ПТО, руководители строительных объектов, собственники строительных компаний

3) Бизнес-проблема (3–5 пунктов):
- План, факт, табели и ресурсы ведутся вручную и разрозненно
- Отсутствует оперативная управленческая картина по объектам и сменам
- Потери времени и денег из-за человеческого фактора и несвоевременных решений
- Нет прозрачной аналитики эффективности смен и сотрудников
- Сложность подготовки отчётов и документов для руководства

4) Результат для бизнеса (3–5 измеримых эффектов):
- Сокращение времени на сбор и обработку данных по сменам
- Повышение прозрачности затрат и эффективности работ
- Снижение ошибок ручного учёта
- Быстрое выявление отклонений план/факт
- Улучшение управленческих решений на основе данных и ИИ-анализа

5) Формат решения:
Telegram-бот + backend AI-логика

6) Каналы ввода данных:
Текст, кнопки, формы, числовые значения

7) Каналы вывода:
Сообщения в Telegram, структурированные отчёты, PDF-документы, управленческие сводки

8) Что ИИ делает в проекте (ровно 3 функции):
1) Анализ план/факт показателей смен
2) Генерация управленческих выводов и инсайтов
3) Подготовка текстовых отчётов и рекомендаций для руководства

9) Какие API/сервисы реально используются сейчас:
Telegram Bot API, OpenAI API, Bitрикс24 REST API

10) Где хранится база/данные:
Bitrix24 (Smart-процессы: объекты, смены, ресурсы, табели)

11) Где развёрнуто:
Backend-разработка в Cursor, серверное развёртывание (VPS)

12) Что уже готово (список функций):
- Telegram-бот для работы со сменами
- Сбор плановых и фактических данных
- Интеграция с Bitrix24
- ИИ-анализ данных и генерация отчётов

13) Что НЕ готово / ограничения (список):
- Масштабирование под несколько компаний
- Расширенная рольвая модель пользователей
- Автоматическая финансовая интеграция с бухгалтерией
